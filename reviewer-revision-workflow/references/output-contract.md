# 输出与归档契约

## 单条审稿意见的交付模板

按以下顺序输出，所有可提交文本均先给英文，后给中文核对译文。

```markdown
### Reviewer comment
[审稿人原文]

### Author's revision plan
[用户确认的修改计划；若处于讨论模式，只分析，不填写后续最终文本]

### Manuscript revision — English (copy-ready)
**Action:** Replace / Insert after / Insert before / Delete

**Location anchor in the English manuscript**
- Preceding text: “...”
- Following text: “...”

**Text to use**
[英文最终文本]

### 稿件修改 — 中文（核对与复制）
**操作：** 替换 / 在后插入 / 在前插入 / 删除

**中文稿定位锚点**
- 修改前文本：“...”
- 修改后文本：“...”

**使用文本**
[中文最终文本]

### Response to Reviewer — English (copy-ready)
Dear Reviewer,

Thank you for this helpful comment. [回复正文；用 **粗体** 标出重点改动。]

**Author action:** In Section X, Page XX, lines XX-XX.

### 给作者核对的中文译文
尊敬的审稿人：

感谢您提出这一宝贵意见。[中文核对译文。]

**作者修改：** 在第 X 节，第 XX 页，第 XX–XX 行。

### Consistency check
[列出稿件与回信中共用的术语、章节、图表或数据；只有发现不一致时，明确标注待确认项。]
```

## 图表辅助回信

如审稿意见需要图表辅助说明，且原稿已有相关图表或作者已提供图表/数据，在英文回信中使用现有编号和可核对的描述，例如：

```markdown
As illustrated in Figure X, [specific observation]. We have clarified this point in the revised manuscript.
```

同时说明图表在稿件中的修改锚点和 `Author action` 占位定位。

## Reviewing memory.md 的追加条目

仅在用户同意归档后，在现有文件末尾追加以下信息，并保持该文件已有标题层级和格式：

```markdown
## Reviewer [编号] — Comment [编号]

- **Issue:** [审稿意见核心]
- **Agreed revision strategy:** [用户确认的策略]
- **Manuscript change:** [英文/中文改动的简要说明和锚点]
- **Response position:** [回信核心表述]
- **Final locator:** In Section X, Page XX, lines XX-XX. [定稿后补充]
- **Reusable note:** [可用于处理后续类似意见的术语、逻辑或表达]
```
